제 1 장
두 피해 유형 — 같은 뿌리, 다른 나무
A · 과세피해 · L2T
원금에 세금이 부과됩니다
원장 → 소득성격 → 과표 → 세액
↑ 이 연결이 없습니다
· 해외 ETF 소득성격 원장 접근 곤란
· 취득원가(Basis) 추적 체계 부재
· 과표 산출 근거 불명확
· 증권사·국세청·기재부·KSD 책임 불명확
· ICI Primary Layout 72년간 미연결
핵심 권리: 과표검증권
B · 조직범죄 송금피해 · T2R
잘못 이동한 돈을 돌려받지 못합니다
송금 → 추적 → 동결 → 반환
↑ 반환 인프라가 없습니다
· 이상거래 탐지 실패
· 지급정지 지연 — 기준 없음
· 대포통장 정보 접근 제한
· 반환 인프라 부재
· 은행·통신사·플랫폼 책임 불명확
핵심 권리: 조직범죄 송금피해 반환권

같은 뿌리: 원장이 없으면 — 약자가 책임자가 됩니다. 소득성격 원장이 없으면 납세자가, 반환 원장이 없으면 피해자가 책임자가 됩니다.

제 2 장
국가책임 7대 평가기준
一
원장성
원장성(元帳性) — 원장이 존재하는가
"권리·과표·송금·자금흐름의 원장이 존재하는가?"
L2T: ICI Primary Layout → 국세청 연결 원장 부재
T2R: 이상거래 탐지 원장 · 지급정지 원장 부재
二
추적성
추적성(追跡性) — 결과에서 원인까지 역추적 가능한가
"세액에서 원장까지, 피해에서 탐지 실패까지 역추적 가능한가?"
L2T: 세액 → 과표 → 소득성격 → ICI Primary Layout
T2R: 피해 → 이체 → 탐지 기준 → 실패 원인
三
설명성
설명가능성 — 국민이 이해할 수 있는 방식으로 설명 가능한가
"왜 이 세금이 부과됐는지, 왜 이 이체가 막히지 않았는지 설명할 수 있는가?"
L2T: 유권해석 한 줄 ≠ 설명가능한 과세
T2R: "고객이 직접 승인했다" ≠ 탐지 의무 설명
四
감사성
감사추적성 — 누가, 언제, 어떤 자료로 판단했는지 기록되는가
"유권해석 작성 시 ICI를 검토했는가? 탐지 로그가 보존되는가?"
L2T: 서면-2024-소득-4836 작성 근거 원장 부재
T2R: 은행 탐지 시스템 작동 로그 비공개
五
복구성
복구성 — 오류·피해 발생 시 정정·환급·반환이 가능한가
"경정청구가 실질적으로 수리되는가? 피해가 실제로 반환되는가?"
L2T: 경정청구 → 기각 → 불복 → 소송 수년
T2R: 구제율 20% 미만 · 평균 6개월
六
책임성
책임성 — 실패 시 책임기관과 책임사업자를 특정할 수 있는가
"기재부인가, 국세청인가, KSD인가? 은행인가, 통신사인가?"
L2T: 5개 기관 책임 분산 → 모두 회피
T2R: 4개 주체 책임 분산 → 피해자만 남음
七
구제성
구제성 — 국민이 실제로 권리구제를 받을 절차가 있는가
"절차가 존재하는가? 실질적으로 작동하는가?"
L2T: 경정청구 절차 있으나 실질 수리율 낮음
T2R: 신고 절차 있으나 구제율 20% 미만

제 4 장
6대 대안 — 지금 만들 수 있습니다
①
정책 신뢰영향평가 도입
신규 금융·세제·AI·데이터 정책에 의무 적용. 7대 기준으로 사전 평가. 적색 등급 정책은 도입 보류.
②
과표검증권 법제화 — L2T
국민은 세액에서 원장까지 역추적할 권리를 가진다. 국세기본법 §81의2 개정. No Tax Without Ledger.
③
송금피해 반환권 법제화 — T2R
조직범죄 피해자는 추적·동결·반환을 요구할 권리를 가진다. 전기통신금융사기 피해방지법 개정. No Return Without Trace.
④
원장책임제 도입
권리별로 원장 관리기관·감독기관·사업자 책임을 법률로 지정한다. 책임이 분산되면 아무도 책임지지 않습니다.
⑤
감사추적성 의무화
모든 공공·금융 데이터 판단에는 감사 가능한 기록을 남긴다. 유권해석 작성 근거, 탐지 로그, 지급정지 소요 시간 포함.
⑥
설명가능한 과세·금융 원칙 도입
국민 재산에 영향을 주는 결정은 설명 가능해야 한다. "고객이 직접 이체했다"는 설명이 아닙니다.
